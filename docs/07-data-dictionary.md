# Data Dictionary

The data dictionary focuses on fields required for business workflow, reporting, and control. It is not a database schema.

## Entity: Credit Application

| Field | Type | Required | Example | Validation / Notes | Owner |
| --- | --- | --- | --- | --- | --- |
| application_id | Text | Yes | BCS-2026-000184 | System generated unique ID. | System |
| customer_id | Text | Yes | CUST-10492 | Links to customer profile. | RM |

| customer_type | List | Yes | Company | Company, Partnership, Sole Proprietor, Other. | RM |
| segment | List | Yes | SME | SME, Commercial, Corporate. | RM |
| rm_owner | Text | Yes | RM-017 | Must be active user. | RM Team |
| application_type | List | Yes | Renewal | New, Renewal, Enhancement, Restructure. | RM |
| status | List | Yes | Pending Credit Review | Controlled status model. | System |
| created_date | Date | Yes | 2026-01-14 | System timestamp. | System |
| submitted_date | Date | No | 2026-01-16 | Populated when submitted. | System |

## Data Quality Rules

| Rule | Description |
| --- | --- |
| DQ-001 | No application can be submitted without at least one facility. |
| DQ-002 | Amount fields must be positive numbers. |
| DQ-003 | Mandatory document items cannot be blank at submission unless a waiver is recorded. |
| DQ-004 | Decision date cannot be earlier than submitted date. |
| DQ-005 | Ready for Facility Setup requires all mandatory conditions to be completed or waived. |
| DQ-006 | Route override requires authorized user, reason, and audit trail entry. |
